소개

Nest는 효율적이고 확장 가능한 Node.js 서버 측 애플리케이션을 구축하기 위한 프레임 워크입니다. 최신 JavaScript를 사용하고 TypeScript(순수 JavaScript와의 호환성은 유지됨) 및 OOP (객체지향 프로그래밍), FP (함수형 프로그래밍) 및 FRP (함수형 반응형 프로그래밍)의 요소를 결합합니다.

조금 들어가면, Nest는 Express를 이용해 만들었습니다. 그러나 많은 다른 라이브러리와의 호환성을 제공합니다. (예. Fastify) 이는 무수히 많은 서드파티 플러그인을 쉽게 사용할 수 있음을 의미합니다.

철학

최근 몇 년 동안 Node.js 덕분에 자바 스크립트는 프론트엔드와 백엔드 애플리케이션 모두를 위한 웹의 "공용어"가 되었습니다. 이로 인해 Angular, React 및 Vue와 같은 멋진 프로젝트가 생겨났습니다. 이 프로젝트는 개발자의 생산성을 향상 시켰습니다. 그리고 빠르고 테스트가 가능하며 확장 가능한 프론트엔드 애플리케이션을 구축할 수 있게되었습니다. 이에 반해 Node (및 서버 측 JavaScript)에는 뛰어난 라이브러리, 헬퍼 및 도구가 많이 있음에도 그 중 아무도 주요 문제인 아키텍처를 효과적으로 해결하지 못했습니다.

Nest는 즉시 테스트 할 수 있고 확장 가능하며 느슨하게 결합되고 쉽게 유지 관리 할 수있는 애플리케이션을 손쉽게 만들 수 있는 아키텍처를 제공합니다.

설치

시작하려면 Nest CLI를 사용하여 기본 프로젝트를 스캐폴딩하십시오.


$ npm i -g @nestjs/cli
$ nest new project-name

Git을 사용 하여 스타터 프로젝트를 설치하십시오.


$ git clone https://github.com/nestjs/typescript-starter.git project
$ cd project
$ npm install
$ npm run start

$ git clone https://github.com/nestjs/javascript-starter.git project
$ cd project
$ npm install
$ npm run start

마지막으로 NPM (또는 Yarn)을 사용 하여 처음부터 새 프로젝트를 시작하십시오.


$ npm i --save @nestjs/core @nestjs/common rxjs reflect-metadata
